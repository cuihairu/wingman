# 项目简介

## 什么是 Wingman？

**Wingman** 是一个游戏自动化可编程控制引擎，基于 C++ 构建，支持 Lua 和 Python 两种脚本语言。

## 设计目标

### 核心目标

1. **高性能** - C++ 核心引擎，Lua/Python 脚本执行，毫秒级响应
2. **安全可靠** - 纯用户态运行，使用合法 Windows API
3. **可编程** - Lua 和 Python 双语言支持，无限扩展可能
4. **易调试** - VS Code 插件支持，断点调试、变量查看、性能分析

### 不做什么

Wingman 遵循单一职责原则，以下功能**不在**项目范围内：

| 功能 | 说明 |
|-----|------|
| ❌ 内核驱动 | 不开发驱动，纯用户态 |
| ❌ 内存读写 | 不读取/修改游戏内存 |
| ❌ DLL 注入 | 不注入代码到游戏进程 |
| ❌ 游戏破解 | 不修改游戏文件/协议 |
| ❌ 分布式网络 | 不做分布式网络 |
| ❌ 账号管理 | 不管理游戏账号 |

## 核心设计原则

### 1. 安全第一

- 纯用户态运行，不进入内核
- 使用合法 Windows API（SendInput 等）
- 不读写游戏内存
- 不注入任何代码到游戏进程
- 模拟真实人类操作模式

### 2. 性能优先

- C++ 实现性能敏感操作
- Lua 和 Python 高效脚本执行
- 零拷贝数据传递
- 批量操作减少跨语言调用

### 3. 多语言支持

- **Lua** - 轻量级、高性能、适合游戏脚本
- **Python** - 生态丰富、适合复杂逻辑和 AI 集成
- 统一 API 设计，两种语言调用方式一致

### 4. 可编程

- 脚本控制，用户可扩展
- 不限制使用场景
- 丰富的 API 暴露
- 支持复杂业务逻辑

### 5. 人性化

- 模拟真实人类操作
- 降低检测风险
- 可配置的"人性化"参数

## 为什么支持两种脚本语言？

### Lua 的优势

- **轻量级** - 极小的内存占用
- **高性能** - 毫秒级启动时间
- **简单** - 语法简洁，易于学习
- **嵌入式** - 天生为嵌入式设计

### Python 的优势

- **生态丰富** - 海量的第三方库
- **AI 生态** - 与 PyTorch、TensorFlow 无缝集成
- **开发效率** - 强大的 IDE 支持和调试工具
- **广泛应用** - 大量开发者熟悉

### 使用建议

- **游戏脚本** - 推荐 Lua，性能优先
- **AI 集成** - 推荐 Python，生态丰富
- **快速原型** - 推荐 Python，开发效率高
- **生产环境** - 根据团队技术栈选择
