# Workspace 演示文档

## 演示目标

展示 Workspace 的核心功能和价值，让观众理解如何通过可视化编排快速搭建游戏管理后台。

## 演示流程

### 1. 介绍背景（2 分钟）

#### 现有问题

- **开发效率低**：每次添加新的管理功能都需要编写大量代码
- **维护成本高**：UI 和业务逻辑耦合，修改界面需要改代码
- **使用门槛高**：非开发人员无法自己创建管理界面

#### 解决方案

- **配置驱动**：所有界面由配置决定，修改配置即可改变界面
- **可视化编排**：通过拖拽和配置，无需编写代码
- **职责分离**：SDK 专注功能注册，Dashboard 专注 UI 编排

#### 核心价值

- ✅ 添加新 Workspace：从几小时 → 几分钟
- ✅ 修改 UI：从改代码 → 改配置
- ✅ 降低门槛：非开发人员也能编排

### 2. 演示现有 Workspace（3 分钟）

#### 步骤 1: 访问玩家管理

1. 打开浏览器，访问 `http://localhost:8000`
2. 登录系统（admin / admin123）
3. 点击左侧菜单"控制台" → "玩家管理"

#### 步骤 2: 展示功能

**Tab 1: 玩家信息**

- 输入玩家 ID：`123456`
- 点击"查询"按钮
- 展示玩家详细信息：
  - 基本信息：ID、昵称、等级、金币
  - VIP 信息：VIP 等级、到期时间
  - 统计信息：登录次数、在线时长
- 点击"更新等级"按钮，演示操作功能

**Tab 2: 玩家列表**

- 展示玩家列表
- 演示分页功能

**关键点**：

- 所有功能都是通过配置实现的

### 3. 演示编排器（5 分钟）

#### 步骤 1: 打开编排器

1. 访问 `/workspace-editor/player`
2. 展示编排器的三栏布局：
   - 左侧：函数列表
   - 中间：布局设计器
   - 右侧：实时预览

#### 步骤 2: 从零开始创建配置

**创建新对象**：

1. 访问 `/workspace-editor/order`（订单管理）
2. 系统自动创建空配置

**添加第一个 Tab**：

1. 点击"添加 Tab"按钮
2. 输入标题："订单列表"
3. 输入图标："ShoppingOutlined"
4. 点击"确定"

**配置布局**：

1. 选择布局类型："列表"
2. 从左侧拖拽函数 `order.list` 到"使用的函数"区域
3. 配置列表函数：输入 `order.list`
4. 右侧实时预览显示列表布局

**添加第二个 Tab**：

1. 点击"添加 Tab"按钮
2. 输入标题："订单详情"
3. 选择布局类型："表单-详情"
4. 拖拽函数 `order.getInfo`
5. 配置查询函数：输入 `order.getInfo`
6. 实时预览显示表单-详情布局

**保存配置**：

1. 点击"保存配置"按钮
2. 显示"保存成功"提示

#### 步骤 3: 使用模板

**演示模板功能**：

1. 点击"使用模板"按钮
2. 展示可用模板：
   - 基础列表
   - 表单-详情
   - 完整管理
3. 选择"完整管理"模板
4. 点击"使用"
5. 配置自动填充，实时预览显示效果

**关键点**：

- 强调拖拽式设计的便捷性
- 强调实时预览的直观性
- 强调模板的快速性

## 常见问题准备

### Q: 性能如何？

**A**: 性能表现优秀：

- 配置缓存 5 分钟，减少 API 调用
- 首屏加载 < 2 秒
- 路由切换 < 500ms
- 配置保存 < 1 秒
- 支持虚拟滚动处理大数据

### Q: 如何扩展？

**A**: 扩展性强：

- 支持自定义布局类型
- 支持自定义渲染方式
- 支持自定义组件
- 提供完整的 API 和文档

### Q: 如何迁移现有页面？

**A**: 迁移简单：

1. 分析现有页面的功能
2. 创建对应的配置
3. 测试验证
4. 逐步替换

不需要一次性迁移，可以逐步进行。

### Q: 安全性如何保证？

**A**: 安全机制完善：

- 权限控制：Workspace、Tab、操作三级权限
- 配置验证：保存前验证配置格式
- 操作审计：记录所有配置变更
- 数据隔离：配置和数据分离存储

### Q: 支持哪些布局类型？

**A**: 目前支持 4 种布局类型：

- **FormDetail**：表单-详情布局
- **List**：列表布局
- **Form**：表单布局
- **Detail**：详情布局

未来会支持更多类型（向导、仪表盘等）。

### Q: 配置可以版本管理吗？

**A**: 目前版本（v1.0）暂不支持配置版本管理，这是 Phase 2 的增强功能。

但可以通过导出配置到 Git 仓库的方式实现版本管理。

### Q: 如何备份和恢复配置？

**A**: 支持配置导入导出：

- 导出：将配置导出为 JSON 文件
- 备份：保存 JSON 文件到安全位置
- 恢复：通过导入功能恢复配置

建议定期备份重要配置。
