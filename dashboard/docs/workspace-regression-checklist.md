# Workspace 回归清单

适用范围：`croupier-dashboard` 对象工作台（Workspaces / WorkspaceEditor / Console）。

## 1. 范围说明

- 本清单覆盖当前产品主路径：函数目录 -> 对象工作台 -> 运行控制台。
- 正式能力优先全量回归，Beta 能力按对象和样板选择性补回归。
- 编辑器预览允许示例数据；控制台与动态运行页必须按真实运行态标准验证。

## 2. 角色维度

- [ ] **管理员**：可查看列表、编辑、保存、发布、取消发布、查看版本、回滚。
- [ ] **编辑者（无发布权限）**：可编辑与保存；发布/取消发布/回滚按钮不可用或不可见。
- [ ] **只读用户**：可查看控制台已发布内容；编辑入口不可用。
- [ ] **无权限用户**：访问管理页或详情页时收到明确错误提示，不出现静默失败。

## 3. 页面维度

### 3.1 Workspaces 列表页

- [ ] 首次加载成功，展示标题、描述、状态、版本、更新时间。
- [ ] 搜索（标题/objectKey/描述）生效，空结果出现空态提示。
- [ ] 状态筛选（全部/草稿/已发布/已归档）生效。
- [ ] 排序（更新时间降序/标题升序）生效。
- [ ] 已归档配置不允许编辑、发布、取消发布。
- [ ] 加载失败时展示错误态，不出现白屏。

### 3.2 WorkspaceEditor 编辑页

- [ ] `objectKey` 非法时给出错误提示。
- [ ] 可新增/编辑/删除/排序 Tab，且默认 Tab 规则始终合法。
- [ ] 正式 / Beta / 实验能力分层展示正确，不出现旧的 `V1 only` 误导口径。
- [ ] 保存前校验可拦截非法配置，并输出可读错误列表。
- [ ] 模板应用后会展示样板摘要、页面骨架、需核对函数与发布前重点。
- [ ] 版本面板可加载版本列表，回滚前显示差异摘要与风险提示。

### 3.3 Console 列表页

- [ ] 仅展示已发布配置。
- [ ] 搜索/排序生效。
- [ ] 无数据时展示空态，失败时展示错误态。
- [ ] 点击卡片可进入详情页。
- [ ] “优先核查 / 稳定入口” 分层正确，评分与风险提示展示正确。

### 3.4 Console 详情页

- [ ] `objectKey` 为空或非法时展示错误提示。
- [ ] 配置不存在时展示错误提示。
- [ ] 渲染器与编辑器预览在同配置下结构一致，但正式态不会静默回落示例内容。
- [ ] 评分、阻塞项、风险项、逐页检查摘要展示正确。

### 3.5 DynamicPage 运行页

- [ ] 加载中、未找到、运行成功三种状态都展示统一运行容器。
- [ ] 当前路由路径可见，异常时可返回上一页并重新加载。
- [ ] 成功态不展示编辑器式示例骨架。

## 4. 发布链路维度

- [ ] 新建配置后可保存草稿并再次打开。
- [ ] 发布成功后，Workspaces 状态与 Console 可见性同步正确。
- [ ] 取消发布后，Console 侧不可见，Workspaces 状态回落正确。
- [ ] 发布前摘要显示对象、标题、质量评分、关键摘要和影响说明。
- [ ] 回滚后可再次保存与发布，历史版本仍可查询。

## 5. 官方样板维度

- [ ] 官方样板入口可从编辑器首页和模板中心进入。
- [ ] 至少核对 1 个运营列表台样板、1 个对象详情台样板、1 个查询-处理工作台样板。
- [ ] 样板应用后摘要、预览区提示与模板详情里的 checklist 保持一致。
- [ ] 套用样板后可继续补字段和函数绑定，不出现主路径中断。

## 6. 治理与可观测性

- [ ] 权限点生效：`read/edit/publish/rollback/delete` 与 UI 操作一致。
- [ ] 关键埋点可观测：页面打开、加载、保存、发布、取消发布、回滚、渲染失败。
- [ ] 渲染失败或加载失败时，用户可见错误提示，且不阻塞页面其他基础交互。

## 7. 执行说明

- [ ] 回归执行顺序：先角色权限，再页面功能，再发布链路，再官方样板，最后版本回滚。
- [ ] 每条失败项必须记录：角色、页面、对象 `objectKey`、操作步骤、错误截图、控制台日志。
- [ ] 回归通过标准：正式主路径 P0/P1 问题为 0，Beta 能力可带风险评估进入下一轮。
